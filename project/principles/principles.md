# Principles

## Operating Principles
1. Core loop first: prioritize the mine, defend, replicate, and launch loop before adding campaign depth.
2. Readable vector gameplay: every important mechanic should be visually legible in vector form.
3. Dogfood Velumin honestly: use Velumin as a downstream dependency rather than relying on private shortcuts.
4. Small MVP, expandable architecture: avoid premature tech trees, maps, complex economies, and broad content systems.
5. Meaningful launch decision: the player's choice of when to launch must affect risk and future state.
6. Present versus future tradeoff: resources spent on parent survival should compete with investment in the child probe.
7. Testable systems: gameplay systems should support deterministic tests where practical.
8. Arcade immediacy: avoid UI and simulation complexity that weakens moment-to-moment play.

## Bootstrap Notes
- These principles are seeded from the supplied Replication Vector design summary.
- Repository evidence is currently limited to `README.md`, `LICENSE`, and `.gitignore`; implementation-specific principles should be validated after code exists.
