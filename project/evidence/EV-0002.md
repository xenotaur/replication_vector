# EV-0002: Velumin Repository Infrastructure Adaptation

## Summary
- Added Velumin-style guidance files, canonical scripts, CI, and a minimal Rust/WASM/Vite skeleton for Replication Vector.
- Recorded the adopted design decision in `project/design/proposals/adopted/DP-0001-velumin-repo-infrastructure.md`.

## Evidence
- Root guidance files now include `AGENTS.md`, `STYLE.md`, and `REVIEWS.md`.
- Canonical scripts now live under `scripts/`.
- CI validation now lives at `.github/workflows/validate.yml`.
- The minimal Rust/WASM crate lives under `replication_vector/`.
- The browser harness lives under `replication_vector/web/`.

## Velumin Dependency Shape
- Velumin is consumed through a scripted Git checkout at `.deps/velumin`.
- `replication_vector/Cargo.toml` depends on `.deps/velumin/webgpu_vector_lib`.
- This shape was selected because the observed Velumin repository stores its Rust manifest under `webgpu_vector_lib/Cargo.toml`, not at the repository root.

## Limitations
- This is an infrastructure skeleton, not a gameplay loop or rendering spike.
- `scripts/develop` is required before validation if `.deps/velumin` or npm dependencies are absent.
